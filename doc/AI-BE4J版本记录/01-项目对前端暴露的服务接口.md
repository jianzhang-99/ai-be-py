# 项目对前端暴露的服务接口

## 文档说明

- 文档目标：梳理原版 `ai-be-master` 对前端实际暴露的接口，供 `ai-be-py` 重构时对照。
- 统计依据：
  - 后端 Controller：`ai-be-master/src/main/java/com/h3y/aibe/manager/controller`
  - 前端调用：`ai-fe-master/src/api`
- 时间基线：按当前仓库源码扫描结果整理。

## 1. 接口分层

原版 AI-BE 实际上有两套暴露方式：

| 层级 | 主要路由前缀 | 用途 |
|------|--------------|------|
| Web 前端/管理后台接口 | `/auth`、`/ai`、`/oss`、`/admin/**` | 供现有 Vue 前端直接调用 |
| 4J/外部接入接口 | `/ai-api/**` | 面向外部系统接入的对外版本 |

当前前端主要使用的是第一套接口；`/ai-api/**` 更像是对外封装版，并未完整覆盖全部前端能力。

## 2. 通用约定

### 2.1 通用响应结构

除 `SSE` 流式接口外，绝大多数接口都返回统一包裹：

```json
{
  "code": 200,
  "data": {},
  "msg": "success"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `number` | 业务状态码，成功时通常为 `200` |
| `data` | `any` | 业务数据 |
| `msg` | `string` | 提示信息 |

### 2.2 分页响应结构

```json
{
  "code": 200,
  "data": {
    "records": [],
    "current": 1,
    "size": 10,
    "total": 100
  },
  "msg": "success"
}
```

### 2.3 认证说明

- 登录成功后返回 `token`
- `sa-token.token-name = satoken`
- 代码中前端默认按登录态访问大部分接口
- `分享接口` 明确支持免登录访问

## 3. 前端业务主链路接口

### 3.1 登录鉴权

| 接口 | 方法 | 说明 | 前端调用情况 |
|------|------|------|--------------|
| `/auth/login` | `POST` | 用户登录 | 已使用 |
| `/auth/logout` | `POST` | 用户登出 | 已使用 |
| `/auth/current` | `GET` | 获取当前用户信息 | 前端已调用，但后端源码中未找到对应实现 |

#### 3.1.1 登录

- 接口：`POST /auth/login`
- 请求体：

```json
{
  "phone": "13800000000",
  "password": "123456"
}
```

- 响应体 `data`：

```json
{
  "userId": 1,
  "phone": "13800000000",
  "token": "satoken_xxx"
}
```

#### 3.1.2 特别说明

- `GET /auth/current` 仅出现在前端代码中，当前 `ai-be-master` 的 `AuthController` 未提供该接口。
- Python 重构时建议明确是否补齐该接口，避免前端保留“幽灵调用”。

### 3.2 AI 对话

| 接口 | 方法 | 说明 |
|------|------|------|
| `/ai/chat` | `POST` | AI 流式对话，返回 `text/event-stream` |

#### 3.2.1 请求体

```json
{
  "sessionId": "web-1711111111111",
  "input": "帮我查一下今天武汉天气",
  "clientType": "web",
  "scene": "QUERY_WEATHER",
  "model": "qwen-turbo",
  "toolEnabled": true,
  "attachments": [
    {
      "ossId": 123,
      "fileName": "图片.png",
      "fileType": "image/png",
      "url": "https://xxx/1.png"
    }
  ],
  "appSource": "WEB"
}
```

关键字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `sessionId` | `string` | 会话 ID |
| `input` | `string` | 用户输入 |
| `scene` | `string` | 场景枚举，如 `QUERY_WEATHER`、`SAVE_ORDER` |
| `model` | `string` | 指定模型 |
| `toolEnabled` | `boolean` | 是否允许工具调用 |
| `attachments` | `array` | 附件列表 |
| `appSource` | `string` | 来源，如 `WEB`、`AIDUN_APP` |

#### 3.2.2 SSE 输出类型

流式输出的每个事件体可抽象为：

```json
{
  "type": "TEXT",
  "content": "正在为你查询天气"
}
```

常见 `type`：

| 类型 | 说明 |
|------|------|
| `THINKING` | 思考过程 |
| `TEXT` | 普通文本 |
| `MARKDOWN` | Markdown 文本 |
| `MAP` | 地图卡片 |
| `CARD_SHIP` | 船舶卡片 |
| `CARD_ORDER` | 运单卡片 |
| `ORDER_LIST` | 运单列表 |
| `CARD_DISPATCH_MONITOR` | 在途监控卡片 |
| `WEATHER` | 天气卡片 |
| `CARD_WATER_LEVEL` | 水位卡片 |
| `SHIP_LIST` | 船舶列表 |
| `CARD_FEEDBACK` | 反馈卡片 |
| `QUICK_QUESTION` | 关联追问 |
| `ERROR` | 错误信息 |
| `DONE` | 结束标记 |

| 接口 | 方法 | 说明 |

#### 3.4.1 语音转文字

- 接口：`POST /ai/input/speech`
- Content-Type：`multipart/form-data`
- 表单字段：`file`
- 返回：识别后的文本

#### 3.4.2 图片 OCR

- 接口：`POST /ai/input/image`
- Content-Type：`multipart/form-data`
- 表单字段：`file`
- 返回：OCR 文本

#### 3.4.3 航线解析

- 接口：`POST /ai/input/shippingRoute`
- Content-Type：`multipart/form-data`
- 表单字段：`file`
- 返回 `ShippingRouteResp`：

```json
{
  "loadAddressName": "武汉阳逻港",
  "loadAddressLon": 114.1234567890,
  "loadAddressLat": 30.1234567890,
  "loadCityName": "武汉",
  "unloadAddressName": "南京龙潭港",
  "unloadAddressLon": 118.1234567890,
  "unloadAddressLat": 32.1234567890,
  "unloadCityName": "南京"
}
```

#### 3.4.4 运单参数解析

该接口在源码里存在两种历史用法：

| 版本 | 入参形态 | 来源 |
|------|----------|------|
| 管理端 Controller | `rawText` 表单参数 | `InputController` |
| 对外 API 版 | JSON `{ "text": "..." }` | `InputApi` |

前端代码当前主要传文字语义，建议 Python 重构时统一成 JSON 版本：

```json
{
  "text": "5月10日武汉装煤5000吨到南京"
}
```

返回 `OrderCardResp`，典型字段：

```json
{
  "orderId": "preview",
  "loadAddressName": "武汉",
  "unloadAddressName": "南京",
  "loadDate": "2026-05-10",
  "goodsInfoName": "煤",
  "goodsCount": 5000,
  "goodsUnit": 1,
  "goodsUnitName": "吨",
  "ships": []
}
```

#### 3.4.5 按 OSS 识别

- 接口：`POST /ai/input/recognizeByOssId?ossId=123`
- 说明：根据 OSS 文件记录自动判断并识别音频/图片

## 5. `/ai-api` 4J 对外接口映射

当前仓库里，4J 版对外接口只覆盖了一部分能力：

| 4J 对外接口 | 内部前端接口 | 说明 |
|-------------|--------------|------|
| `/ai-api/chat` | `/ai/chat` | 对外 AI 流式对话 |
| `/ai-api/chat/history/page` | `/ai/chat/history/page` | 对外历史分页 |
| `/ai-api/chat/history/listBySessionId/{sessionId}` | `/ai/chat/history/listBySessionId/{sessionId}` | 对外历史详情 |
| `/ai-api/chat/history/deleteBySessionId/{sessionId}` | `/ai/chat/history/deleteBySessionId/{sessionId}` | 对外删除历史 |
| `/ai-api/input/speech` | `/ai/input/speech` | 语音转文字 |
| `/ai-api/input/image` | `/ai/input/image` | 图片 OCR |
| `/ai-api/input/file` | `/ai/input/file` | 文件占位处理 |
| `/ai-api/input/shippingRoute` | `/ai/input/shippingRoute` | 航线解析 |
| `/ai-api/input/analysisOrderParams` | `/ai/input/analysisOrderParams` | 运单参数解析 |
| `/ai-api/input/recognizeByOssId` | `/ai/input/recognizeByOssId` | 按 OSS 识别 |
| `/ai-api/oss/upload` | `/oss/upload` | OSS 上传 |
| `/ai-api/oss/{id}` | `/oss/{id}` | OSS 查询 |

未见 `/ai-api` 对外版的能力：

- 登录鉴权 `/auth/**`
- 运单创建 `/ai/order/create`
- 管理后台 `/admin/**`
- 分享查询 `/ai/chat/history/share/**`

## 6. 对 Python 重构的建议

### 6.1 建议优先保留的兼容接口

- `/ai/chat`
- `/ai/chat/history/*`
- `/ai/input/image`
- `/ai/input/recognizeByOssId`
- `/ai/input/analysisOrderParams`
- `/oss/upload`
- `/ai/order/create`

### 6.2 建议补齐或统一的点

- 补齐或删除 `/auth/current`
- 统一 `/ai/input/analysisOrderParams` 的入参格式，建议只保留 JSON 版本
- 明确 `/ai` 与 `/ai-api` 的职责边界，避免两套路由长期并存
