# 阶段设计

## 说明

本目录用于沉淀 AI-BE-PY 的分阶段落地方案。

项目当前不是从零开始写一份“大而全”的最终架构文档，而是按照：

1. 先跑起来
2. 先打通最小业务闭环
3. 再逐步补外部系统、RAG、记忆、可观测性

的节奏推进。

## 阶段划分原则

### 第一阶段

目标是 `能启动`、`能对话`、`能识别意图`、`至少能跑通 2~3 个真实业务场景`。

### 第二阶段

目标是 `补业务深度`，把找船、查船、运单录入、天气、知识问答做成稳定可演示版本。

### 第三阶段

目标是 `可持续运行`，补充持久化、缓存、RAG、记忆、观测、测试、部署能力。
