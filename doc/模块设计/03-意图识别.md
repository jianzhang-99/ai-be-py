# 03-意图识别

## 模块职责

识别用户当前对话的意图，确定进入哪个业务场景。

## 对应原项目

- `IntentRecognizer.java`
- `IntentRuleGateway.java`
- `IntentRules.java`
- `LlmIntentClassifier.java`
- `FinalIntentResolver.java`

## 三层识别策略

```
┌─────────────────────────────────────────┐
│           用户输入                        │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Layer 1: 规则网关 (Rule Gateway)         │
│  - 关键词直命中                          │
│  - 条件找船识别                          │
│  - 运单续接识别                          │
└─────────────────┬───────────────────────┘
                  ↓ (未命中)
┌─────────────────────────────────────────┐
│  Layer 2: LLM 分类 (LLM Classifier)      │
│  - 使用 DeepSeek/OpenAI                 │
│  - 输出场景枚举                          │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  Layer 3: 守卫校正 (Guard)               │
│  - 安全兜底                              │
│  - 异常纠正                              │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│           SceneEnum 输出                 │
└─────────────────────────────────────────┘
```

## 规则网关规则

### 规则类型

1. **关键词直命中**
   - "天气" → QUERY_WEATHER
   - "查船" → QUERY_SHIP
   - "找船" → FIND_SHIP

2. **条件找船识别**
   - 检测航线条件（起运港、目的港）
   - 检测货物信息（货类、吨位）

3. **运单续接识别**
   - 上轮是 SAVE_ORDER 且状态为 WAITING_USER

4. **地点查询识别**
   - 港口城市名识别

5. **订单修改识别**
   - 字段补充类输入

### 状态定义

`backend/graph/state/agent_state.py`

```python
from typing import TypedDict, Optional, List
from pydantic import BaseModel

class IntentInfo(BaseModel):
    """意图信息"""
    intent: str  # 意图名称
    confidence: float  # 置信度
    method: str  # 识别方式: rule/llm/guard

class AgentState(TypedDict):
    """Agent 状态"""
    user_input: str
    user_id: str
    session_id: str

    # 意图识别结果
    intent: Optional[IntentInfo]
    current_scene: Optional[str]

    # 上下文
    messages: List[str]
    working_memory: dict

    # 工具调用
    tool_result: Optional[dict]

    # RAG
    retrieved_docs: List[str]

    # 输出
    response: Optional[str]
```

### 意图节点

`backend/graph/nodes/intent_node.py`

```python
from backend.services.intent_service import IntentService
from backend.agents.registry import AgentRegistry

async def intent_node(state: AgentState) -> AgentState:
    """
    意图识别节点

    三层识别：规则 → LLM → 守卫
    """
    user_input = state["user_input"]
    working_memory = state.get("working_memory", {})

    intent_service = IntentService()

    # Layer 1: 规则网关
    intent = intent_service.rule_gateway.recognize(user_input, working_memory)
    if intent:
        return {"intent": IntentInfo(
            intent=intent,
            confidence=1.0,
            method="rule"
        )}

    # Layer 2: LLM 分类
    intent = await intent_service.llm_classifier.classify(
        user_input,
        working_memory
    )
    if intent:
        return {"intent": IntentInfo(
            intent=intent,
            confidence=0.8,
            method="llm"
        )}

    # Layer 3: 守卫校正（默认直接返回）
    return {"intent": IntentInfo(
        intent="TALK",
        confidence=0.5,
        method="guard"
    )}
```

### 规则网关

`backend/services/intent_service.py`

```python
from typing import Optional

class IntentRuleGateway:
    """规则网关"""

    KEYWORD_RULES = {
        "天气": "QUERY_WEATHER",
        "查船": "QUERY_SHIP",
        "找船": "FIND_SHIP",
        "运价": "QUERY_FREIGHT",
        "录单": "SAVE_ORDER",
        "查单": "QUERY_ORDER",
    }

    def recognize(self, user_input: str, working_memory: dict) -> Optional[str]:
        """
        规则识别
        """
        # 1. 关键词直命中
        for keyword, intent in self.KEYWORD_RULES.items():
            if keyword in user_input:
                return intent

        # 2. 找船续接判断
        if working_memory.get("current_scene") == "FIND_SHIP":
            if working_memory.get("state") == "WAITING_USER":
                return "FIND_SHIP"

        # 3. 运单续接判断
        if working_memory.get("current_scene") == "SAVE_ORDER":
            if working_memory.get("state") == "WAITING_USER":
                return "SAVE_ORDER"

        return None
```

## 设计取舍

### 优点
- 规则优先：高频意图零成本匹配
- 模型兜底：复杂意图可推理
- 可测试：规则可单元测试

### 代价
- 规则需要持续维护和优先级排序

### 风险
- 规则和 prompt 同时修改可能造成意图漂移
