## 设计原则

### 1. 分层解耦

```
API Layer     →  接收请求，返回响应
    ↓
Services     →  业务编排，调度下层
    ↓
Graph/Nodes  →  LangGraph 状态机
    ↓
Infra        →  外部依赖（LLM/DB/Cache）
```

### 2. 配置外置

所有配置通过 `config.py` 和 `.env` 管理，不硬编码。

### 3. 可测试性

每个模块独立，可单独测试。核心逻辑都有单元测试。

### 4. 事件驱动

长时间任务通过 Workers 异步处理，不阻塞主流程。

## 核心模块说明

### Intent Recognition（意图识别）

**三层意图识别**：
1. **规则网关** → 关键词高频匹配
2. **LLM分类** → 模型推理
3. **守卫校正** → 安全兜底

### Scene Execution（场景执行）

每个场景对应一个独立工作流，场景间解耦：

- `doc_qa` - 文档问答
- `find_ship` - 找船
- `save_order` - 运单录入
- `query_order` - 运单查询
- `query_ship` - 船舶查询
- `query_freight` - 运价查询
- `query_weather` - 天气查询

### Memory（记忆系统）

**三层记忆**：
1. **L1 近期对话** - chat_log 原始轮次
2. **L2 对话摘要** - chat_summary 压缩
3. **L3 用户画像** - user_memory_profile 跨会话

### RAG（知识增强）

**检索流程**：
1. Query 改写（融入历史）
2. Embedding
3. 向量检索（Milvus）
4. Rerank
5. 阈值过滤
6. 知识注入
