# AI-BE-PY 项目结构设计

## 一、项目概述

AI-BE-PY 是 AI-BE（Java SpringAI 版本）的 Python 重构版本，基于 LangGraph 构建的航运领域智能对话微服务。

### 核心定位
- **语言**: Python 3.11+
- **框架**: FastAPI + LangGraph
- **目标**: 航运垂直领域的 AI Agent 服务

### 与 SpringAI 版本对比

| 维度 | SpringAI 版本 | Python 版本 |
|------|---------------|-------------|
| 编排方式 | FlowEngineProcessor | LangGraph |
| 状态管理 | SceneContext | Graph State |
| 工具调用 | IToolService | Tools + MCP |
| 扩展性 | 中 | 高（Python 生态） |
| 学习成本 | 中 | 低 |

## 三、模块说明

### 1. API 层 (`api/`)

**职责**: 对外提供 HTTP/ SSE 接口，接收用户请求，返回响应。

| 文件 | 说明 |
|------|------|
| `deps.py` | 依赖注入：数据库连接、Redis、LLM 客户端等 |
| `routers/chat.py` | 对话接口，支持 SSE 流式响应 |
| `routers/health.py` | 健康检查接口 |

### 2. Agent 层 (`agents/`)

**职责**: 多 Agent 管理，Agent 工厂模式。

| 文件 | 说明 |
|------|------|
| `factory.py` | Agent 工厂，创建不同类型 Agent |
| `registry.py` | Agent 注册表，运行时动态加载 |

### 3. 工具层 (`tools/`)

**职责**: 封装外部工具调用，包括 MCP 协议和业务工具。

| 文件/目录 | 说明 |
|----------|------|
| `registry.py` | 工具注册中心，所有工具在此注册 |
| `mcp/mcp_client.py` | MCP 协议客户端 |
| `weather.py` | 天气查询工具 |
| `order.py` | 运单查询/录入工具 |
| `ship.py` | 船舶查询工具 |

### 4. Graph 层 (`graph/`)

**职责**: LangGraph 状态机核心，定义节点、边和工作流。

| 目录 | 说明 |
|------|------|
| `state/` | 状态定义（Agent State） |
| `nodes/` | 图节点（意图识别、路由、工具调用等） |
| `workflows/` | 工作流定义（对话主流程 + 场景工作流） |

### 5. Services 层 (`services/`)

**职责**: 业务逻辑层，编排下层组件。

| 文件 | 说明 |
|------|------|
| `intent_service.py` | 意图识别服务 |
| `scene_service.py` | 场景调度服务 |
| `session_service.py` | 会话管理服务 |

### 6. Infra 层 (`infra/`)

**职责**: 基础设施抽象，解耦外部依赖。

| 目录 | 说明 |
|------|------|
| `llm/` | LLM 适配（支持 DeepSeek、OpenAI 等） |
| `vector/` | 向量数据库（Milvus） |
| `cache/` | Redis 缓存 |
| `storage/` | OSS 对象存储 |

### 7. Workers 层 (`workers/`)

**职责**: 异步任务处理（如长任务后台处理）。

## 四、数据流向

```
用户请求
    ↓
[API Layer] api/routers/chat.py
    ↓
[Services] session_service.py (会话管理)
    ↓
[Graph] chat_workflow.py (LangGraph 主流程)
    ├── [Node] intent_node.py (意图识别)
    ├── [Node] routing_node.py (路由决策)
    ├── [Node] tool_node.py (工具调用)
    ├── [Node] rag_node.py (RAG 检索)
    ├── [Node] memory_node.py (记忆读写)
    └── [Node] response_node.py (响应生成)
    ↓
[Infra] llm/ (LLM 调用)
    ↓
SSE 流式响应
```

## 七、命名规范

- **目录**: 小写下划线命名（如 `scene_workflows`）
- **Python 文件**: 小写下划线命名（如 `intent_node.py`）
- **类名**: 大驼峰命名（如 `AgentState`）
- **函数**: 小写下划线命名（如 `get_intent`）
- **常量**: 全大写下划线命名（如 `MAX_RETRIES`）
