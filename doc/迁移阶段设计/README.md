# 阶段设计

## 说明

本目录用于沉淀 AI-BE-PY 的分阶段落地方案。

项目当前不是从零开始写一份"大而全"的最终架构文档，而是按照：

1. 先跑起来
2. 先打通最小业务闭环
3. 再逐步补外部系统、RAG、记忆、可观测性

的节奏推进。

## 阶段划分

### 第一阶段：能跑能演示（已完成）

**目标：** 服务可启动、主链路闭环、至少 2~3 个航运业务场景可演示。

**核心交付：**
- FastAPI + LangGraph 主流程
- SSE 流式对话
- 意图识别（TALK / QUERY_WEATHER / QUERY_SHIP / SAVE_ORDER）
- 静态工具注册机制
- `.env` 配置加载

**文档：** `01-第一阶段MVP技术方案.md`、`02-第一阶段开发任务拆解.md`、`03-第一阶段接口与验收.md`

---

### 第二阶段：补业务深度（进行中）

**目标：** 真实业务系统接入、持久化落库、会话管理、工具稳定性增强。

**核心交付：**
- MySQL 持久化（sys_user、chat_session、chat_message）
- Redis 会话缓存
- 运吨吨 Pilot API 接入（运单预录入）
- 大数据 API 接入（找船查询）
- 动态工具注册 + 熔断降级 + 超时控制
- 基础结构化日志

**文档：** `04-第二阶段技术方案.md`

---

### 第三阶段：可持续运行

**目标：** 生产级稳定性、RAG 检索、多层记忆、完整可观测性、测试体系和自动化部署。

**核心交付：**
- Milvus 向量数据库 + RAG 检索增强
- 三层记忆系统（短期 / 中期 / 长期）
- Prometheus 指标 + OpenTelemetry 链路追踪
- RABC 权限体系
- 单元测试覆盖率 > 80%、集成测试完整覆盖
- Docker + Docker Compose 部署

**文档：** `05-第三阶段技术方案.md`

---

## 完成定义速查

| 阶段 | 关键验收项 |
|------|-----------|
| 第一阶段 | 服务启动 + `/health` + `/api/chat/stream` 流式返回 + 至少 2 个场景可演示 |
| 第二阶段 | Session 持久化 + Redis 缓存 + Pilot/BigData API 接入 + 工具熔断降级 |
| 第三阶段 | RAG 问答可用 + 多层记忆正常 + 指标暴露 + 测试覆盖率 > 80% + Docker 可部署 |
