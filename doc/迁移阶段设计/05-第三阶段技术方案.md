# 第三阶段：可持续运行

## 1. 背景与目标

前两阶段已建立核心链路和业务深度。第三阶段目标是让系统具备生产级稳定性、可持续迭代能力，包括 RAG 检索、多层记忆、可观测性、测试体系完善和自动化部署。

## 2. 阶段目标

### 2.1 业务目标

- 航运知识库智能问答（RAG）
- 多轮会话记忆（短期 + 长期）
- 运营数据报表（会话统计、意图分布）
- 完整用户权限体系（RABC）

### 2.2 技术目标

- Milvus 向量数据库接入（知识检索）
- 多层记忆系统（短期会话 + 长期用户画像）
- 完整可观测性（日志 + 指标 + 链路追踪）
- 自动化测试体系（单元 + 集成 + E2E）
- CI/CD 流水线

## 5. RAG 设计

### 5.1 知识库类型

| 类型 | 内容 | 来源 |
|------|------|------|
| 运价知识 | 航线运价、附加费规则 | 大数据 API / Excel 导入 |
| 港口知识 | 港口介绍、装卸货要求 | 文档 |
| 货物知识 | 货物分类、危品规则 | 文档 |
| 船型知识 | 船型分类、适货要求 | 文档 |

### 5.2 文档处理流程

```text
原始文档 (PDF/Word/Excel)
  -> 文本提取
  -> 滑动窗口切分 (chunk_size=512, overlap=64)
  -> Embedding 生成 (text-embedding-3-small)
  -> Milvus 向量存储
  -> 元数据索引 (type, source, create_time)
```

### 5.3 检索链路

```python
async def retrieve_knowledge(query: str, top_k: int = 5) -> list[Document]:
    # 1. 查询向量化和相似检索
    query_embedding = await embedding_service.embed(query)
    results = await milvus_client.search(
        collection_name="航运知识库",
        vector=query_embedding,
        top_k=top_k,
        filter={"type": "运价知识"},  # 可选过滤
    )
    # 2. Rerank（可选）
    reranked = await rerank_service.rerank(query, results)
    return reranked
```

### 5.4 RAG 增强节点

在 LangGraph 中增加 `rag_node`：

```python
class RAGNode:
    async def execute(self, state: AgentState) -> AgentState:
        query = state["user_input"]
        docs = await self.rag_service.retrieve_knowledge(query, top_k=3)

        # 构造 context
        context = "\n".join([doc.content for doc in docs])

        # 构造增强 prompt
        enhanced_prompt = f"""根据以下知识回答问题：
        {context}

        问题：{query}
        """
        state["enhanced_prompt"] = enhanced_prompt
        return state
```

## 6. 记忆系统设计

### 6.1 三层记忆架构

```
短期记忆 (Short-term)
  - 当前会话内的消息历史
  - LangGraph State 内管理
  - 会话结束清除

中期记忆 (Medium-term)
  - 用户画像和偏好
  - Redis Hash 存储，TTL=30天
  - key: user:{user_id}:profile

长期记忆 (Long-term)
  - 跨会话重要信息（历史运单、常用货物）
  - MySQL 存储
  - 定期总结写入（每周）
```

### 6.2 记忆读写流程

```python
class MemoryManager:
    async def load_context(self, user_id: str, session_id: str) -> dict:
        """加载当前会话所需的所有记忆"""
        short = await self.short_term.get(session_id)
        medium = await self.medium_term.get(user_id)
        long = await self.long_term.get(user_id)
        return {
            "recent_history": short,
            "user_profile": medium,
            "persistent_info": long,
        }

    async def save_after_turn(
        self,
        user_id: str,
        session_id: str,
        user_input: str,
        assistant_output: str,
        intent: str,
    ):
        """每轮对话后自动保存"""
        await self.short_term.append(session_id, user_input, assistant_output)
        if intent in ["QUERY_SHIP", "SAVE_ORDER"]:
            await self.medium_term.update_preference(user_id, intent)

    async def weekly_summary(self, user_id: str):
        """每周总结写入长期记忆"""
        recent = await self.short_term.get_recent(user_id, days=7)
        summary = await self.llm.summarize(recent)
        await self.long_term.save(user_id, summary)
```

### 8.2 权限装饰器

```python
def require_permission(code: str):
    """接口权限校验装饰器"""
    def decorator(func):
        @wraps(func)
        async def wrapper(request, *args, **kwargs):
            user = request.state.current_user
            if not has_permission(user.role_keys, code):
                raise PermissionDenied(code)
            return await func(request, *args, **kwargs)
        return wrapper
    return decorator

@router.post("/api/order/submit")
@require_permission("order:submit")
async def submit_order(...): ...
```

## 9. 测试体系设计

### 9.1 测试分层

```
单元测试 (Unit)
  - 每个函数 / 类的孤立测试
  - Mock 所有外部依赖
  - 覆盖率目标: > 80%

集成测试 (Integration)
  - 模块间交互测试
  - 使用 TestClient 测 API 层
  - 使用 Stub 替代真实 DB / 外部服务

端到端测试 (E2E)
  - 完整用户路径测试
  - 使用真实（或模拟）的外部系统
```

### 9.2 关键场景测试用例

| 场景 | 测试内容 |
|------|----------|
| 登录 | 正确凭证登录成功 / 错误密码拒绝 / 用户不存在 |
| 对话主链路 | 流式输出完整 / 中途异常处理 |
| 工具调用 | 成功 / 超时 / 熔断降级 |
| RAG | 检索到相关文档 / 无结果时降级 |
| 记忆 | 多轮后上下文正确 / 记忆写入读取 |
| 权限 | 有权限用户正常访问 / 无权限用户 403 |
