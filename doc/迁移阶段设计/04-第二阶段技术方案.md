# 第二阶段：补业务深度

## 1. 背景与目标

第一阶段已验证核心链路可跑。第二阶段目标是补齐真实业务深度，把找船、查船、运单、天气等场景做成稳定可演示版本，同时建立持久化和外部系统集成能力。

## 2. 阶段目标

### 2.1 业务目标

- 找船列表查询（航线 + 船型 + 载重筛选）
- 船舶详细信息查询（MMSI、位置、到港预报）
- 运单信息抽取 + 外部系统预录入
- 天气服务稳定接入（支持多城市）
- 会话状态持久化（Redis 会话缓存）

### 2.2 技术目标

- MySQL 持久化落库（sys_user、chat_session、chat_message）
- Redis 会话缓存
- 外部业务系统适配（运吨吨 Pilot API、大数据 API）
- 工具注册与发现机制增强
- 基础可观测性（日志 + 调用链追踪）

## 3. 阶段范围

### 3.1 范围内

#### 持久化

- `sys_user` 表接入（登录、用户信息）
- `chat_session` 会话表（session_id、user_id、create_time）
- `chat_message` 消息表（session_id、role、content、intent、create_time）

#### 外部系统集成

- 运吨吨 Pilot API（运单预录入）
- 大数据 API（找船查询）
- 天气 API 稳定接入

#### 会话管理

- Redis Session 存储（token → user_id）
- 会话历史写入数据库

#### 工具增强

- 动态工具注册（registry + factory）
- 工具超时控制
- 工具熔断降级

### 3.2 范围外

- Milvus RAG 检索
- 多轮记忆系统
- 完整权限体系（RABC）
- 运营管理后台

## 5. 数据模型

### 5.1 新增表

```sql
-- chat_session
CREATE TABLE chat_session (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    session_id    VARCHAR(64) NOT NULL UNIQUE,
    user_id       BIGINT NOT NULL,
    scene         VARCHAR(32) DEFAULT 'DEFAULT',
    status        TINYINT DEFAULT 1,           -- 1=活跃, 0=结束
    create_time   DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time   DATETIME ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_create_time (create_time)
);

-- chat_message
CREATE TABLE chat_message (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    session_id    VARCHAR(64) NOT NULL,
    role          VARCHAR(16) NOT NULL,          -- user/assistant/system
    content       TEXT,
    intent        VARCHAR(32),
    tool_name     VARCHAR(64),
    tool_result   TEXT,
    latency_ms    INT,
    create_time   DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_session_id (session_id),
    INDEX idx_create_time (create_time)
);
```

## 7. 工具改造

### 7.1 动态注册增强

```python
class ToolRegistry:
    def register(self, tool: BaseTool) -> None: ...
    def register_factory(self, name: str, factory: Callable) -> None: ...
    def get(self, name: str) -> BaseTool | None: ...
    def list_tools(self) -> list[str]: ...
```

### 7.2 工具熔断降级

```python
class CircuitBreaker:
    def __init__(self, threshold: int = 3, timeout: int = 60): ...

    async def call(self, func: Callable, *args, **kwargs):
        if self._state == "open":
            raise ToolDegradedError("circuit open")
        try:
            return await func(*args, **kwargs)
        except Exception:
            self._failures += 1
            if self._failures >= self._threshold:
                self._state = "open"
            raise
```

### 7.3 工具超时控制

每个工具执行增加 `asyncio.wait_for(timeout=30)`，超时抛 `ToolTimeoutError`。

## 9. 可观测性

### 9.1 结构化日志

```python
import structlog
structlog.configure(
    processors=[
        structlog.processors.add_log_level,
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.processors.JSONRenderer(),
    ]
)
```

### 9.2 必记日志字段

| 字段 | 说明 |
|------|------|
| `request_id` | 单次请求唯一标识 |
| `session_id` | 会话标识 |
| `user_id` | 用户标识 |
| `intent` | 识别的意图 |
| `tool_name` | 工具名 |
| `latency_ms` | 耗时（LLM、Tool、Total） |
| `error` | 错误信息 |

### 9.3 慢操作告警

- LLM 调用 > 30s → 告警
- 工具调用 > 15s → 告警
- 整体响应 > 60s → 告警
